# Смоук-тест proxyapi (Claude Sonnet 5) — 2026-07-22

Этап 2, сессия 1 (TASK_STAGE2.md, пункт 1). Один живой вызов на каждую
роль нового пайплайна через proxyapi.ru, модель `claude-sonnet-5`, тема
«Порядок действий в примерах» (историческая проблемная тема — на GigaChat
регулярно давала брак, см. Этап 1).

## Файлы

- `1_content.json` — вызов 1 (генератор контента).
- `2_questions.json` — вызов 2 (генератор вопросов), raw-дампы всех шагов/quiz.
- `3_verdict.json` — вызов 3 (судья) на первом проверяемом вопросе.

## Результат

Вся арифметика в сгенерированном тексте и вопросах — **верна** (проверено
глазами: `(5+3)×2=16`, `4+6×3=22`, `24÷4×2=12`, `3×(8-5)+12÷4=12` — все
значения посчитаны правильно). Судья согласился с автором на первом
проверяемом вопросе (`author_correct=0`, `verifier_correct_index=0`,
`accepted=true`).

## Технический баг, найденный и исправленный по ходу

Первый прогон упал с `KeyError: 'text'` — модель (через этот прокси)
возвращает `content` как массив блоков, и для более длинных промптов
ПЕРЕД текстовым блоком идёт блок `{"type": "thinking", ...}` (extended
thinking). Код в `llm_client.py` брал `content[0]["text"]` по фиксированному
индексу — не работает, если блоков несколько. Исправлено: ищем блок по
`type == "text"`, а не по индексу. См. `app/services/llm_client.py`,
`_proxyapi_generate()`.

## Стоимость

Не измерена точно (не сохранил `usage` из ответов API в этом прогоне) —
для справки короткий debug-вызов на 23 входных/9 выходных токенов
подтвердил формат ответа. Три полных вызова (контент + вопросы + один
вердикт судьи) на этой теме почти наверняка укладываются в лоу-single-digit
рубли по тарифам Sonnet 5 (800₽/4500₽ за 1M вход/выход) — на порядок
меньше расчётного бюджета ~210₽ на полный 10-темный замер. В сессии 3
(полный замер) — сохранять `usage` из каждого ответа для точного учёта
фактических затрат (пункт «стоимость фактическая vs расчётная» в
TASK_STAGE2.md).
